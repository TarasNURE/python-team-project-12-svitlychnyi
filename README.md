# Практичне заняття №6: Основи командної розробки Python-проєктів

## Опис проєкту

У цьому репозиторії представлені результати виконання практичної роботи №6. Метою проєкту є демонстрація навичок командної розробки, використання системи контролю версій Git та організації Python-коду за модульним принципом.

---

## Завдання 1. Консольний аналізатор даних

Програма зчитує числові дані з файлу, виконує їх аналіз та виводить основні статистичні показники.

### Структура проєкту

* `main.py` — головний файл запуску програми;
* `io_utils.py` — модуль для роботи з файлами (читання та запис даних);
* `analysis.py` — модуль для виконання статистичного аналізу.

### Запуск програми

```bash
python main.py
```

---

## Завдання 2. Індивідуальне завдання (Варіант 12): Аналізатор лог-файлів

Програма автоматично аналізує системні журнали (логи), визначає загальну кількість записів, а також підраховує кількість повідомлень типу `ERROR` та `WARNING`.

### Структура проєкту

* `main_logs.py` — головний файл запуску аналізатора логів;
* `io_utils.py` — модуль для зчитування лог-файлів;
* `analysis.py` — модуль для пошуку та аналізу записів.

### Запуск програми

```bash
python main_logs.py
```

### Важливо

Для коректної роботи програми в кореневій директорії проєкту повинен знаходитися файл:

```text
logs.txt
```

Під час виконання роботи було дотримано принципів модульності та командної розробки програмного забезпечення. Усі зміни коду відстежувалися за допомогою системи контролю версій Git.

* створення структури проєкту;
