# Практичне заняття 6: Основи командної розробки Python-проєктів

Цей репозиторій містить результати виконання практичної роботи, що демонструє навички командної розробки, використання системи контролю версій Git та модульної структури проєкту.

## 1. Базове завдання: «Консольний аналізатор даних»
Проєкт реалізує зчитування числових даних із файлу, їх обробку та обчислення базової статистичної інформації (середнє значення, мінімум, максимум, стандартне відхилення).

- **Структура проєкту:**
    - `main.py` — головний модуль інтеграції.
    - `io_utils.py` — модуль для вводу та виводу даних.
    - `analysis.py` — модуль статистичного аналізу.
- **Як запустити:**
  ```bash
  python main.py
2. Індивідуальне завдання (Варіант 12): «Аналізатор лог-файлів»
Проєкт призначений для автоматизованого аналізу системних логів. Програма підраховує загальну кількість записів, а також виявляє кількість критичних помилок (ERROR) та попереджень (WARNING).

Структура проєкту:

main_logs.py — головний скрипт для керування процесом аналізу.

io_utils.py — модуль для зчитування лог-файлів.

analysis.py — модуль для аналізу текстових записів.

Як запустити:

Bash
python main_logs.py
(Примітка: для роботи програми у папці має знаходитися файл logs.txt)
