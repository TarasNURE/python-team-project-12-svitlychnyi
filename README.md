# Практичне заняття №6

## Основи командної розробки Python-проєктів

У межах цієї практичної роботи було створено Python-проєкт із використанням модульної структури та системи контролю версій Git. Основна мета роботи — закріпити навички командної розробки, організації коду та роботи з репозиторіями.

## Базове завдання — Консольний аналізатор даних

Програма зчитує числові дані з файлу, виконує їх аналіз та виводить основні статистичні показники.

### Файли проєкту

* `main.py` — запуск програми;
* `io_utils.py` — робота з файлами;
* `analysis.py` — обчислення статистичних показників.

### Запуск

```bash
python main.py
```

## Індивідуальне завдання (Варіант 12) — Аналізатор лог-файлів

Програма призначена для аналізу системних логів. Вона визначає загальну кількість записів, а також підраховує кількість повідомлень типу `ERROR` і `WARNING`.

### Використані файли

* `main_logs.py` — запуск аналізатора;
* `io_utils.py` — зчитування лог-файлів;
* `analysis.py` — аналіз даних журналу подій.

### Запуск

```bash
python main.py
```

Перед запуском необхідно переконатися, що в кореневій папці проєкту знаходиться файл `logs.txt`.

## Розподіл ролей

Під час виконання роботи використовувалися принципи командної розробки.

**Team Lead / Integrator**

* створення структури проєкту;
* організація взаємодії між модулями;
* налаштування роботи з файлами.

**Developer / Tester**

* реалізація логіки аналізу;
* перевірка коректності роботи програм;
* тестування результатів.
